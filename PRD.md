# PRD: HomePilot

## 1. 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | HomePilot |
| 한 줄 설명 | 지역별 아파트 시세와 주변 데이터를 분석하여 실거주 및 구매 의사결정을 돕는 플랫폼 |
| 대상 유저 | 첫 내 집 마련 또는 전세를 준비하는 사람 |

## 2. 배경 및 문제 정의

내 집 마련이나 전세를 처음 준비하는 사람은 부동산 실거래가, 시세 변동, 교통·생활 인프라 등 흩어져 있는 정보를 스스로 수집하고 비교·해석해야 하는 부담을 안고 있다. 관련 지식과 경험이 부족한 상태에서 다수의 지역·단지를 동시에 비교하기 어렵고, 데이터를 봐도 이것이 좋은 선택인지 판단하기 힘들다.

HomePilot은 AI를 활용해 자연어 질의만으로 조건에 맞는 지역·아파트를 탐색하고, 복잡한 데이터를 이해하기 쉬운 리포트로 요약하며, 관심 매물을 저장·비교할 수 있게 하여 의사결정 부담을 낮춘다.

## 3. 목표 (Goals)

- 사용자가 부동산 전문 지식 없이도 자연어로 원하는 조건의 지역/아파트를 탐색할 수 있게 한다.
- 실거래가, 시세, 교통, 인프라 등 파편화된 정보를 AI가 요약한 리포트로 제공하여 의사결정 시간을 단축한다.
- 여러 후보 지역/단지를 한 화면에서 비교할 수 있도록 하여 선택의 확신을 높인다.

### Non-goals

- 매물 중개, 계약, 대출 알선 등 거래 자체를 지원하지 않는다.
- 상업용 부동산, 토지, 재건축/재개발 투자 분석은 초기 범위에서 제외한다.

## 4. 핵심 기능

### 4.1 AI 지역·아파트 검색

사용자가 자연어로 원하는 조건을 입력하면 AI가 조건을 이해하여 적합한 지역 또는 아파트를 탐색한다.

**예시 질의**
- "성수동에서 8억 이하 아파트 추천해줘."
- "신혼부부가 살기 좋은 지역 알려줘."
- "걸어서 15분 이내에 초등학교가 있는 아파트 중 서울에서 7억이하의 아파트를 알려줘"

**요구사항**
- 홈페이지 최상단(Above the fold)에 자연어 검색창을 배치하여, 별도 페이지 이동 없이 랜딩 즉시 검색이 가능하도록 한다.
- 자연어 질의에서 지역명, 예산, 평형, 생활 조건(신혼부부, 학군, 직장 접근성 등) 등 핵심 조건을 추출한다.
- 조건이 모호하거나 불충분할 경우 추가 질문을 통해 조건을 구체화한다.
- 검색 결과는 조건 충족도 순으로 정렬된 지역/아파트 리스트와 지도 뷰로 제공한다.
- 비교 질의("A와 B 중 어디가 좋을까")의 경우 두 후보를 나란히 비교하는 응답을 제공한다.

### 4.2 AI 부동산 분석 리포트

선택한 아파트 또는 지역의 정보를 AI가 이해하기 쉬운 형태로 요약한다.

**포함 정보**
- 실거래가 추이
- 최근 시세 변화
- 주변 단지 비교
- 교통 접근성
- 생활 인프라 (학교, 병원, 상권, 공원 등)
- 장점 및 고려사항 (AI 종합 코멘트)

**요구사항**
- 공공 부동산 API 및 실거래가 데이터를 기반으로 최신 데이터를 반영한다.
- 수치 데이터는 차트/그래프로 시각화하고, 해석은 AI가 자연어로 요약한다.
- 리포트는 지역 단위와 개별 단지 단위 모두 생성 가능해야 한다.

### 4.3 관심 지역 저장 및 비교

사용자가 관심 있는 지역 또는 아파트를 저장하고 여러 단지를 비교할 수 있다.

**비교 항목**
- 평균 시세
- 최근 가격 변동
- 교통
- 생활 인프라
- AI 종합 의견

**요구사항**
- 로그인한 사용자는 관심 지역/단지를 리스트에 저장할 수 있다.
- 저장된 항목 중 2개 이상 선택 시 비교표(테이블/카드) 형태로 항목별 비교를 제공한다.
- 비교 결과 하단에 AI가 종합하여 "어떤 조건에서 어떤 선택이 유리한지"를 코멘트로 제공한다.

## 5. 사용자 플로우 (예시)

1. 사용자가 검색창에 자연어로 조건을 입력한다.
2. AI가 조건을 해석해 후보 지역/아파트 리스트와 지도를 보여준다.
3. 사용자가 특정 지역/아파트를 클릭하면 AI 분석 리포트 페이지로 이동한다.
4. 리포트를 확인 후 관심 목록에 저장한다.
5. 관심 목록에서 2개 이상 항목을 선택해 비교 화면으로 이동, 비교표와 AI 종합 의견을 확인한다.

## 6. 기술 스택

| 영역 | 기술 |
|---|---|
| Frontend | Next.js, React, TypeScript, Tailwind CSS |
| Backend | Supabase |
| AI | OpenAI API (GPT) 또는 Claude API |
| Data | 공공 부동산 API, 실거래가 데이터, 지도 API (카카오 또는 네이버) |
| Deployment | Vercel |
| Analytics | PostHog, Google Analytics 4 |

## 7. 디자인 방향

전체적으로 신뢰감을 주는 컬러와 분위기를 기본 톤으로 하여, 다음 원칙에 따라 디자인한다.

- 신뢰감 있는 컬러 팔레트와 분위기를 전체 UI의 기본 톤앤매너로 적용 (금융·부동산 서비스로서의 신뢰감 우선)
- 미니멀, 모노톤 컬러 기반
- SaaS 대시보드 스타일
- 지도 중심 UI 구성
- 데이터 카드 기반 정보 구조
- Apple + Linear 스타일의 절제된 인터랙션과 타이포그래피
- 여백을 적극 활용한 레이아웃

## 8. 성공 지표 (제안)

- 자연어 검색 사용률: 검색 세션 중 자연어 검색 기능 사용 비율
- 리포트 조회율: 검색 결과에서 리포트 상세 진입 비율
- 관심 목록 저장률: 리포트 조회 대비 저장 전환율
- 비교 기능 사용률: 관심 목록 2개 이상 보유 사용자 중 비교 기능 사용 비율
- 재방문율 (WAU/MAU)

## 9. 향후 고려사항 (Out of scope for v1)

- 대출 한도 계산 등 금융 연동 기능
- 알림 기능 (관심 단지 시세 변동 알림)
- 모바일 앱 (초기에는 반응형 웹 우선)
